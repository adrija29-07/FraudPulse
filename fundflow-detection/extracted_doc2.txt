1. Executive Summary
Money laundering through banking systems costs the global economy trillions annually. Traditional rule-based fraud detection systems fail to catch sophisticated layering schemes that spread transactions across many accounts, branches, and channels. Fund flow patterns that appear benign in isolation reveal criminal intent only when viewed as a connected graph.
This PRD defines an intelligent Fund Flow Tracking system that maps end-to-end movement of funds within the bank using graph analytics and machine learning. The system visualises suspicious patterns, enables investigators to trace complete money trails, and auto-generates evidence packages for reporting to the Financial Intelligence Unit (FIU).
2. Problem Statement
2.1 Background
Banks process millions of transactions daily across accounts, products, branches, and digital channels. Money launderers exploit the complexity and volume of these transactions to obscure the origin and destination of illicit funds through a three-stage process: Placement, Layering, and Integration.

5. Functional Requirements
5.1 Transaction Ingestion & Graph Construction
Ingest real-time transaction data from core banking, NEFT/RTGS/IMPS, UPI, and ATM channels
Construct a live transaction graph: nodes = accounts/entities, edges = fund transfers with amount, timestamp, channel
Enrich nodes with KYC data: customer type, declared income, occupation, risk rating
Maintain rolling 12-month transaction history in graph database
Support batch ingestion for historical data analysis
5.2 Pattern Detection Engine
Graph Neural Network (GNN) model trained on labelled fraud cases to score suspicious subgraphs
Rule-based layer for deterministic patterns: structuring below Rs.50,000, same-day round trips, dormant activation
Anomaly detection for accounts deviating from their historical behavioural baseline
Community detection algorithms (Louvain, Label Propagation) to identify money mule networks
Velocity checks: flag accounts with >20 transactions per hour
5.3 Investigation Workbench
Interactive force-directed graph: zoom, pan, expand node connections
Timeline view: animate fund flow step-by-step to replay the money trail
Filter controls: by date range, amount, account type, channel, branch
Node annotations: investigators can add notes and flag nodes as confirmed fraud
Shortest path tool: trace the quickest route between any two accounts
Evidence collection: select nodes/edges and bundle into exportable case file
5.4 Risk Scoring & Alert Queue
Each suspicious pattern assigned a risk score (0-100) based on pattern type, amount, and account profiles
Priority alert queue sorted by risk score with SLA timers
Alert categories: Suspicious Transaction Report (STR), Cash Transaction Report (CTR), counterfeit
One-click escalation to senior investigator or legal team
5.5 FIU Evidence Package Generation
Auto-generate pre-filled STR (Suspicious Transaction Report) in FIU-IND prescribed format
Attach transaction graph snapshot, account details, and behavioural analysis
Include chronological narrative of fund movement auto-written by LLM
Digital audit trail of all investigator actions on the case
Secure submission portal integration with FIU-IND goAML system
