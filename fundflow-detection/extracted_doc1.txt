1. Design Overview & Inspiration
The PS3 Fund Flow Tracking platform draws from two distinct design languages seen in the uploaded reference screenshots — blending them strategically across the two primary user modes of the application.

1.1  Reference Analysis — Image 1 (Dark Finance Dashboard)
The first reference shows a sophisticated crypto/finance dashboard with a near-black background (#0D0D0D), burnt-orange accent color (#E8660A), and modular card-based layout. Key design observations:
Dark theme creates authority and focus — ideal for the high-stakes Alert & Monitoring view
Orange accent (#E8660A) used exclusively for CTAs, positive indicators, and active elements — e.g. the 'Exchange' button, percentage badges, and chart highlights
Cards are dark (#1A1A1A) with subtle border separation — no heavy drop shadows
Typography uses a bold sans-serif for large numbers (account balances, profit figures) with smaller muted labels below each metric
The Portfolio Tokens section uses pill/chip components for category tags (similar to how we tag account types: Savings, Current, Corporate)
Activity chart uses an area/line chart with orange fill — maps directly to our transaction volume over time chart
'Withers' badge pattern (orange pill) will be adapted for alert severity tags: HIGH, MEDIUM, LOW

1.2  Reference Analysis — Image 2 (AceDeal Light Dashboard)
The second reference shows a light-mode analytics dashboard with a soft purple/lavender color scheme. This maps to the Investigator Workbench — where analysts review cases in detail. Key design observations:
Light cards with very subtle purple-tinted backgrounds (#F7F5FF) reduce eye strain during long investigation sessions
The Balance History panel layout (label + amount + timestamp + status) maps directly to our Transaction Timeline panel
Payment Analytics section with 'Orders Placed: 120' and metric cards maps to our 'Alerts Triggered', 'Amount at Risk', 'Cases Resolved' KPI cards
The pie/donut chart (Total Orders by Category) maps to our Fund Flow by Channel breakdown (NEFT, RTGS, UPI, ATM)
'In Progress' and 'Paid' status pills map to our case status workflow: Open, Under Investigation, Escalated, Closed
Recent Orders list with avatar + description + status maps to our Recent Alerts feed with account name + pattern description + risk score

3.1  Number Formatting Standards
Large financial figures are displayed in the same high-contrast monospace style as Image 1 ($5,654.32 in large bold):
Amounts above ₹1 lakh: use Indian number format — ₹12,54,320.00
Amounts displayed in graph tooltips: abbreviate — ₹12.5L, ₹3.2Cr
Percentage changes: always show sign (+2.32%, -4.65%) with color — green positive, red negative (exact pattern from Image 1 badges)
Risk scores: displayed as XX/100 in monospace font with gradient color coding
Timestamps: relative ('2 mins ago') in the feed; absolute (DD MMM YYYY HH:MM) in the detail view

5.2  Screen 2 — Transaction Graph Deep Dive
The visual centrepiece. Full-screen dark graph with controls panel on the left. This is the most memorable demo moment.
Background: #0D0D0D with subtle radial gradient (#1A1A2E at centre)
Node colors: Green (#00C48C) = clean account, Orange (#F5A623) = suspicious, Red (#FF4757) = confirmed fraud, Grey (#555577) = dormant
Node size: proportional to total transaction volume through that account
Edge colors: Animated blue (#4F9EFF) for live/recent flows; static grey for historical
Edge thickness: proportional to transfer amount
On node click: side panel slides in (right) showing account profile, KYC data, full transaction history, and risk score gauge
Control strip (top): Filter by date range | Filter by minimum amount | Highlight only flagged paths | Play/pause live stream

10. Accessibility Guidelines
Even for a hackathon demo, basic accessibility signals professionalism to judges.

Guideline
Requirement
Implementation
Color Contrast
WCAG AA minimum (4.5:1 for text)
All text on dark cards meets contrast. Orange (#E8660A) on black exceeds 4.5:1.
Risk Color Redundancy
Never use color alone to convey risk
Every risk badge includes both color AND text label: HIGH, MEDIUM, LOW
Focus States
Visible keyboard focus ring
Custom focus ring in orange: outline 2px solid #E8660A, offset 2px
Graph Accessibility
Provide table alternative
Alert Queue panel serves as accessible text alternative to the visual graph
Font Size
Minimum 12px for all readable text
Smallest text in design is 11px for badge labels — acceptable for dashboard context
Loading States
Indicate async operations
Skeleton shimmer on cards while data loads. Spinner on report generation.
