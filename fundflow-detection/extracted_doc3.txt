1. Why Typography Matters for This Dashboard
The PS3 Fund Flow Tracking platform handles two very different cognitive tasks at the same time. Analysts need to read large financial figures at a glance under pressure, and also read dense investigative text and table data during long case review sessions. A single font cannot serve both jobs equally well.

The four-font stack below was chosen by matching visual patterns directly observed in the reference screenshots — the dark finance dashboard and the AceDeal light UI — and mapping each font to a specific cognitive role in the interface.

It signals that this is a serious, professional platform — not a generic data tool.

2.2  Syne — What NOT to Do
Never use Syne below 13px — its geometric forms lose legibility at small sizes
Never use Syne Regular (400) — the light weight loses its identity entirely
Never use Syne for body paragraphs or table content — use DM Sans instead
Never use Syne for financial numbers — Space Mono must handle all numeric data

2.3  Google Fonts Import — Syne Only

font-family: 'Syne', sans-serif;

3. Font 2 — Space Mono (Financial Numbers)

₹5,654.32
Space Mono 700 — Numbers
Space Mono is a fixed-width typeface designed for code and data display. Every digit is exactly the same width — this prevents the number '₹1,234.56' from jumping in width when it updates live to '₹10,000.00'. This is the exact visual treatment used in Image 1 for the '$5,654.32' account balance and '$32,569.32' total profit figure.

3.2  Number Formatting Standards
Indian number format for all INR amounts: ₹12,54,320.00 (not ₹1,254,320.00)
Abbreviated format in graph tooltips and small cards: ₹12.5L, ₹3.2Cr, ₹450K
Always show sign for delta values: +₹2,340.00 or −₹890.00
Risk scores always shown as XX / 100 with a space around the slash
Percentages always show two decimal places: 56.32% not 56%
Never mix Space Mono and Syne in the same number — all digits must be Space Mono

3.3  Google Fonts Import — Space Mono Only
@import url('https://fonts.googleapis.com/css2?family=Space+Mono:wght@400;700&display=swap');

font-family: 'Space Mono', monospace;

font-variant-numeric: tabular-nums;  /* Critical for live updates */

4. Font 3 — DM Sans (Body & Labels)

DM Sans 400 — Body
DM Sans is a geometric sans-serif specifically optimised for UI and digital screens. It maintains excellent legibility at 12–15px — the size range used for all table content, card descriptions, timestamps, and form labels. Its slightly condensed proportions allow more content density without crowding, matching the tight information layout seen in both reference dashboards.

4.2  DM Sans — What NOT to Do

5. Font 4 — DM Mono (Badges, Tags & IDs)

HIGH RISK
DM Mono 500 — Badge
DM Mono is the monospaced companion to DM Sans — designed by the same foundry to work together. It is used for all pill badges (HIGH / MEDIUM / LOW risk), status tags (OPEN / IN REVIEW / CLOSED), account IDs, transaction hashes, and any code-like content. Its tight tracking and consistent character widths make pill labels perfectly balanced inside their containers, matching the 'Withers' badge style in Image 1 and the 'In Progress' status pill in Image 2.

6. Complete Google Fonts Import
Single import line — loads all four fonts in one HTTP request. Place this at the top of your CSS file or inside the <head> of your HTML.

6.1  Single-Line Import
@import url('https://fonts.googleapis.com/css2?family=Syne:wght@700;800&family=Space+Mono:wght@400;700&family=DM+Sans:wght@300;400;500&family=DM+Mono:wght@400;500&display=swap');

6.2  CSS Custom Properties (Root Variables)
Paste these into your :root block. Reference these variables everywhere — never hard-code font-family strings in component styles.

:root {
  /* Font Families */
  --font-display:  'Syne', sans-serif;
  --font-number:   'Space Mono', monospace;
  --font-body:     'DM Sans', sans-serif;
  --font-badge:    'DM Mono', monospace;

  /* Type Scale */
  --text-hero:     60px;
  --text-h1:       36px;
  --text-h2:       26px;
  --text-h3:       20px;
  --text-metric-xl:52px;
  --text-metric-lg:40px;
  --text-metric-md:28px;
  --text-body-lg:  15px;
  --text-body:     14px;
  --text-body-sm:  13px;
  --text-label:    12px;
  --text-badge:    10px;
  --text-caption:  11px;

  /* Font Weights */
  --weight-black:  800;
  --weight-bold:   700;
  --weight-medium: 500;
  --weight-regular:400;
  --weight-light:  300;
}

8. Tailwind CSS Configuration
If your team is using Tailwind CSS (recommended), extend the theme with these tokens in tailwind.config.js:

// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      fontFamily: {
        display: ['Syne', 'sans-serif'],
        number:  ['Space Mono', 'monospace'],
        body:    ['DM Sans', 'sans-serif'],
        badge:   ['DM Mono', 'monospace'],
      },
      fontSize: {
        'hero':    ['60px',  { lineHeight:'1',   letterSpacing:'-2px'  }],
        'h1':      ['36px',  { lineHeight:'1.1', letterSpacing:'-1px'  }],
        'h2':      ['26px',  { lineHeight:'1.2', letterSpacing:'-0.5px'}],
        'metric':  ['40px',  { lineHeight:'1',   letterSpacing:'-1px'  }],
        'metric-sm':['28px', { lineHeight:'1.1', letterSpacing:'-0.5px'}],
        'body':    ['14px',  { lineHeight:'1.6', letterSpacing:'0'     }],
        'body-sm': ['13px',  { lineHeight:'1.5', letterSpacing:'0'     }],
        'label':   ['12px',  { lineHeight:'1.4', letterSpacing:'0'     }],

9.2  Anti-Patterns — Never Do This
Anti-Pattern
Why It Breaks
Correct Fix
Using Arial or Inter instead of DM Sans
Generic look — loses the fintech identity entirely
Replace with DM Sans 400 at same size
Using Syne for body paragraphs
Syne at 13px becomes illegible; its geometric forms need space
Use DM Sans 400 for all body text
Using DM Sans for financial amounts
Proportional spacing causes layout shift on live updates
Use Space Mono with tabular-nums always
Using Space Mono for headings
Too heavy and mechanical for display — loses elegance
Use Syne 800 for all display/heading text
Mixing Space Mono and DM Mono in same number
Inconsistent width rhythm — looks broken
Pick one: Space Mono for numbers, DM Mono for badges only
Using font-weight 400 for Syne
Thin weight loses all visual authority
Always use 700 or 800 for Syne
Setting letter-spacing on Space Mono numbers
Monospace already has fixed spacing — adding more looks wrong
Use letter-spacing: 0 or -1px only for Space Mono
